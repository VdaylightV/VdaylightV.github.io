视觉感知要素

人眼

结构

  • 虹膜：控制进入眼的光量
  • 视网膜：物体在视网膜上成像
  • 晶状体：光线通过晶状体进入眼睛，晶状体可调节焦距
  • 中央凹(concavity)：视网膜上对颜色/细节敏感的区域

特点

  • 成像焦距：14-17mm

  • 亮度适应：

  • 人眼可感知的光强度范围很广——暗视觉和亮视觉两种不同感知细胞的感光范围所构成的感光范围很广

  • 人眼可以同时分辨的强度范围有限——给定条件下，仅有一定的感知范围，超出的部分是无法区分颜色的

    人眼的错觉和欺骗性

  • 马赫带效应：亮度条带交界处发生灰度的上冲和下冲

  • 与不同亮度对比：与不同亮度对比后，人眼对同一亮度的感知有所不同

  • 视错觉：平行，长短

    电磁波谱

  • $能量E=普朗克常数h×频率\nu$

    物体的颜色：

  • 物体均匀反射：呈白色

  • 反射特定光：呈该颜色

    单色光：

  • 没有颜色的光

  • 唯一属性：强度

  • 对单色光的感知：黑色（最暗）、灰色、白色（最亮）

  • 灰度：表示强度的数值

  • 单色图即灰度图像

    彩色光：

  • 范围：0.43-0.79mm的电磁波

    图像形成模型

  • 二维函数$f(x,y)=i(x,y)r(x,y):0<f(x,y)<\infty$

  • 入射分量：$0<i(x,y)<\infty$——表征入射光强度
  • 反射分量：$0<r(x,y)<1$——表征反射率

  • 投射系数：表征光穿透物体的程度

    灰度范围

  • 单色图像的灰度值：$L_{min} \le l \le L_{max}$

    ​ 最小值：$L_{min}=i_{min}r_{min}$

    ​ 最大值：$L_{max}=i_{max}r_{max}$

  • 灰度范围：

    • [$L_{min},L_{max}$]
    • 归一化：[0, L-1]
    • 0表示黑色，$L-1$表示白色，其余为灰色

图像取样和量化

基本概念

  • 目的：把连续图像转换为数字图像
  • 取样：对坐标进行数字化
  • 量化：对幅值进行数字化

表示数字图像

  • 二维数组$f(x,y)$:

    二维数组中每个元素的值为像素

    • 包含$M$行，$N$列
    • $x$=0,1,2,…,$M$-1
    • $y$=0,1,2,…,$N$-1
    • $f(x,y)$表示在$(x,y)$处的值

  • 空间域

    • 图像坐标所张成的实平面

  • 空间坐标

    • $x$和$y$

  • 灰度级数$L=2^k,k\in Z$

    • 灰度值$[0,L-1]$内的整数

  • 动态范围：

    • $\frac{最大可检测灰度}{最小可检测灰度}$

  • 图像的对比度：

    • 图像内最高和最低灰度之间的差值
    • 动态范围光、对比度高

存储数字图像

  • 存储数字图像的比特数:$b=M×N×k$
  • $k$比特图像：图像有$2^k$个灰度级别时，称为$k$比特图像

分辨率

  • 空间分辨率：

    • 图像中可辨别的最小细节的度量
    • 单位距离线对数
    • 单位距离点数
      • 单位英尺点数（dpi）
    • 空间单位
      • 图像像素大小并不能表示清晰程度
      • 像素大小可反映设备成像能力
    • 空间分辨率过低：图片失真

  • 灰度分辨率

    • 灰度级别中可体现的最小变化
      • 灰度级别通常是2的整数次幂
    • 用比特数表示灰度分辨率
      • 256个灰度级别意味着8比特分辨率
    • 比特数过低，灰度分辨率过低：出现伪轮廓

  • 空间、灰度分辨率的联合影响：

    • 等偏爱曲线：

      • 沿等偏爱曲线，人对画质的感受不变
      • 图像所需细节越多，需要的比特数越少（伪轮廓被大量细节掩盖）
      • 增加空间分辨率，可适当降低比特数

  • Background Cluster
  • Intraclass variation

It is such a difficult and challenging problem for computer science.

Attempts:

  • Huble and Wisel: Edge Detection

  • Data-Driven Approach:

    • Collect a dataset of images and labels
    • Use Machine Learning to train a classifier
    • Evaluate the classifier on new images

  • First classifier: Nearest Neighbor

    • train: Memorize all data and labels
    • test: Predict the label of the most similar training image
    • Dataset: CIFAR10
    • Distance Metric to compare images：
      • L1 Distance: Manhattan distance: $d_1(I_1,I_2)=\sum|I_1^p,I_2^p|$. If you know some features have significant meaning. This has coordinate-dependent.
      • L2 Distance: Euclidean distance: $d_2(I_1,I_2)=\sqrt{\sum(I_1^p-I_2^p)^2}$If you don’t know the meaning of the features

  • K-Nearest Neighbors:

    • Take a majority vote among the K closest points.

    • This can smooth the decision boundary.

    • White region: no majority, just randomly choose a label.

    • The choices:

      • the values of K

      • the best distance

      • The above two are hyperparameters.

    • How to set the hyperparameters:

      • #Idea1: Choose hyperparameters that work best on the data—-a very bad idea.
      • #Idea2: Split data into train and test, choose hyperparameters that work best on test data—-no idea how algorithm will perform on new data
      • #Idea3: Split data into train, validation, and test, choose hyperparameters on validation and evaluate on test. Train on the training data, evaluate the model on validation data and pick the set of hyperparameters that performs best on the validation sets. Finally, take the best classifier to work on the test set, the accuracy is the ones that should be on the report or paper.
      • #Idea4: Cross-validation: Split the training data into several folds. Each time we select a fold as a validation set and use others as training data. After a cycle , we calculate the average of the accuracy to represent the performance and select the hyperparameters.
    • The K-Nearest Neighbor on images never used.
      • Very slow.
      • L1 or L2 distance is not a good way to represent pictures.
      • Curse of dimensionality —- exponetial.

#Dynamic Programming:动态规划

动态规划问题概述：

  • 动态规划用于在约束条件下找到问题的最优解，其思路为首先解决子问题，再逐步解决大问题。
  • 能用动态规划求解问题的前提是：每个子问题都是独立且离散的，不能存在依赖关系
  • 每种动态规划解决方案都涉及网络，网络的单元格中的值通常就是需要优化的值。
  • 每个单元格都是问题的子问题，可通过将问题分解为子问题来考虑网络的坐标轴。

背包问题：

  • 问题描述：小偷的背包可装下4磅物品，现有如下三样物品可供盗窃

    • 音响(S)：$3000, 4磅
    • 笔记本电脑(L)：$2000, 3磅
    • 吉他(G)：$1500, 1磅

  • 问题解决：

    • 构建网络，横坐标为背包容量（1磅到4磅）,纵坐标为商品。

    • 按行填充网络

      | | 1 | 2 | 3 | 4 |
      | ———- | ———- | ———- | ———- | —————— |
      | 吉他(G) | $1500 G | $1500 G | $1500 G | $1500 G |
      | 音响(S) | $1500 G | $1500 G | $1500 G | $3000 S |
      | 笔电(L) | $1500 G | $1500 G | $2000 L | $3500 LG |

    • 单元格计算方法：Max(上一个单元格的值, 装入当前商品的价值+剩余空间的价值)

    • 由网格得出最优解为电脑+吉他。

动态规划网络的特点及要求：

  • 行的排列顺序不影响最终结果。
  • 若对于可取走一部分的商品，如大米，豆类等散称物品，动态规划无法求解（贪婪算法可求解）。
  • 横坐标中子背包的容量粒度取决于质量最小的商品。

最长公子串问题：

  • 问题描述：求解两个序列中最长的连续子串。

  • 问题解决：

    • 网格坐标：分别为两个子串

    • 单元格计算：若单元格所在横纵坐标的子串相同，则单元格值为右上角单元格值加1，否则为0。

    • 结果：最长子串长度为网格所有单元格值中的最大值。

最长公共子序列问题：

  • 问题描述：求解两个序列中的最长公共子序列（除去了连续的要求）

  • 问题解决：

    • 网格坐标：分别为两个子串

    • 单元格计算：若单元格所在横纵坐标的子串相同，则单元格值为右上角单元格值加1，否则为该单元格左边和上边单元格值中较大的那一个。

    • 结果：最长公共子序列长度为网格所有单元格值中的最大值。

#Class4-Opening the Box Model

Display

  • How the elements are displayed is determined by their display property, including block,inline,inline-block and none.

Box Model

  • In HTML&CSS, each element is considered as a rectangular box.

  • The box model has the following properties:

    • width
    • height
    • padding
    • border
    • margin

  • The calculation of the total width or height:

    $margin*2 + border*2+padding*2+width/height$

Some Tips:

  • The width and height only make sense to non-inline elements, because the inline elements need to accommodate their content and size.

  • The padding and margin both leave some blank space. The padding is affiliated to the element itself and shares the background with the element, while the margin is affiliated to the parental element of the element and shares the background with the parent’s background.

  • The margins only work horizontally—-left and right on inline-level elements. Meanwhile, the paddings work on all sides of inline-level elements. Both of them work normally on non-inline elements.

  • Some ways to declare margin or padding.

    • The borders have similar properties. However, the have the properties of styles(solide, dashed, dotted, none) and colors(e.g. #949599).

    • We can also round the corners of the border:

    • There are also some ways for us to redefine the calculation of our box model, which is called box-sizing.

      • content box: the value we set is just the size of the content box, we must add the length and width of margin, padding and border.
      • padding box: the value we set includes the size of padding, so we just need to add the length and width of margin and border.
      • border box: the value we set is just the size of our element and we just need to add the length and width of margin.

    • To choose the box-sizing, sometimes we need to add the vendor prefix to support browser-specific properties & values in CSS3.

#Greedy Algorithm： 贪婪算法

贪婪算法的相关概念

算法思路
  • 每一步选择局部最优解
  • 最终得到全局最优解
Example：
  • 旅游问题：每次选择可在剩余时间内旅游完的价值量最高的景点。——非最优解
  • 卡车存储空间问题：每次选择卡车剩余空间中可装载的最大的物品。——非最优解

近似算法

贪婪算法解不可解问题
  • 对于某些问题，随着元素的数目增多，运算量会急剧增加从而导致不可解。
  • 通常采用贪婪算法作为近似算法（Approximation Algorithm）去得到非常接近的解。
近似算法的评价标准
  • 速度有多快
  • 得到的解与最优解的接近程度

集合覆盖问题

广播节目覆盖问题：
  • 问题描述：有许多个电台（n），每个电台覆盖一定数量的州，想要举办一个广播节目让全美所有州都能收听，求最小需要投放广播节目的电台集合。
  • 基本思路：列出所有可能的广播台的集合，计算他们所覆盖的州，找到能覆盖所有州的最小广播台的集合。
  • 存在的困难：对于n个电台而言，有$2^n$种可能的集合数目，随着电台数目增加，所需的计算量呈指数增长，时间复杂度为$O(2^n)$。
  • 贪婪算法的近似：每次选择一个能覆盖最多未覆盖州的电台，直到覆盖所有州。

旅行商问题

  • 问题描述：一个旅行商要前往几个城市，同时要确保时间最短，如何规划行程。

  • 基本思路：列出所有可能的线路并计算其花费时间，选择花费时间最短的那一条线路。

  • 存在的困难：随着城市数目增加，例如$n$个城市，共有$n!$种不同的线路，计算量急剧增加，算法的时间复杂度为$O(n!)$。

  • 贪婪算法的近似：任选一起点，每一步选择距离最近的城市前往。

  • 其他补充说明：

  • 对于同一城市之间的往返，其花费时间可能不同。
    • 对于图算法中的最短路径算法，若指定需要经过某几个节点也便转化成了旅行商问题。

NP完全问题

  • 简单定义：以难解著称的问题。

  • 补充说明：NP完全问题没有办法准确判别，但可根据以下蛛丝马迹判断：

    • 元素较少时，运算速度很快，元素数量增加时，速度变得非常慢。
    • 涉及“所有组合”的问题，通常是NP完全问题。
    • 不能将问题分成小问题，必须考虑各种可能的情况，这可能是NP完全问题。
    • 问题涉及序列且难以解决，这可能是NP完全问题。
    • 问题涉及集合且难以解决，这可能是NP完全问题。
    • 问题可转换为集合覆盖或旅行商问题，肯定是NP完全问题。

#Dijkstra’s Algorithm:狄克斯特拉算法

加权图

  • 对于每一条边都有一个对应的数字的图称为加权图。
  • 每一条边对应的数字称为权重。
  • 计算非加权图的最短路径可使用广度优先搜索，对于加权图的最短路径，应使用狄克斯特拉算法

环

  • 在有向图中，两个节点相互指向则会成环，相当于无向图中两个节点的连接（无向图意味着两个节点互相指向对方，其实就是环）。
  • 在有向图的路径中，若包含环则会导致路径增加，因最短路径。
  • 狄克斯特拉算法仅适用于有向无环图（Direct Acyclic Graph， DAG）
  • 对于存在负权边的图，狄克斯特拉算法也不适用，应选用贝尔曼-福德算法（Bellman-Ford Algorithm）。

狄克斯特拉算法

算法思想：
  1. 找出最便宜的节点，即在最短时间内前往的节点。
  2. 对于该节点的邻居节点，检查是否有前往他们的更短路径，若有，就更新其开销
  3. 重复上述过程，对图中的每一个节点都执行上述操作
  4. 回溯，计算最终路径

找到途中最便宜的节点，并确保没有到该节点更便宜的路径。

需要的数据结构
  • 图的表示（Graph）——散列表：包含每一条边的权值
  • 已有开销（Costs）——散列表：从初始节点开始，未知开销均初始化为$+\infin$
  • 父节点（Parents）——散列表：用于记录路径，便于回溯追踪起点到终点的路径
  • 已探索集（Closed）——数组：用于记录已探索过的，即已找到最便宜路径的节点

Common CSS Property Values

Four ways to represent colors:

Hexadecimal Colors

  • We use six bit of hexadecimal digit to represent the color, the first two bits stand for the red channel, the last two bits stand for blue channel and the middle two bits stand for green channel.
  • If the two bits are the same in the same channel, we can just use three bits of hexadecimal digit to represent them.

RGB & RGBa Colors

  • In RGB Colors, we just rewrite the representation of the color to the following format: rgb(###, ###, ###), where ### stands for a number between 0~255.
  • In RGBa Colors, we add an fourth value $\alpha$, whose number is between 0-1, which stands for the transparency of the color.

HSL & HSLa Colors

  • In HSL Colors, we still have three values. However, the first one —hue, is between 0-360, which identifies the degree of a color on the color wheel. The second one—saturation is between 0~100%, which identifies how saturated with color the hue is. The third one—lightness, identifies how dark or light the hue value is, which is between 0~100% with 0 stands for completely black and 100% stands for completely white. The format is this: hsl(###, ###, ###)
  • In HSLa Colors, we just add a fourth value which stands for the transparency of the color.

The ways to represent length

  • There are two different ways to represent length in page or website——Absolute Length & Relative Length

Absolute Lengths

  • The simplest length representation. The length are fixed to a physical measurement
  • Pixel is the most common one, which is noted as px.
  • This kind of representation lacks flexibility among different display devices.

Relative Lengths

  • The relative lengths are determined according to some other fixed length.

Percentages

  • The length of the element is determined by their parent element.
  • This representation is very helpful in designing a web page’s layout.

Em

  • The length is calculated by comparison of the font size.
  • This is very useful in designing a textbox.
