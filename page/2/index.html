#Class2:Getting to Know HTML

Basic Semantics

Overview

Semantics:

  • The semantics in HTML gives the content of the page meaning of defines the structure of the page.

Divisions & Spans

Commons:

  • <div>s <span>s are both generic containers which has no overarching meaning.
  • <div>s <span>s are often used to apply targeted styles to a contained set of content.

Differences:

  • A <div> is a block-level element that is commonly used to identify large groupings of content, and which helps to build a web page’s layout and design.
  • A <span>, on the other hand, is an inline-level element commonly used to identify smaller groupings of text within a block-level element.

Block vs. Inline Elements

Headings

  • Headings are block-level elements which have six different ranks:

  • This ranks shows the order of the content and their relevance.

Paragraphs

  • Paragraphs are block-level elements which store some information of the website.

Text: Bold

Two ways:

  • <strong>: makes text bold and place a strong importance on it.
  • <b>: just makes the text bold to show them in a different style —— stylistically offset, with no intent to draw intention.

Text: Italicize

Two ways:

  • <em>: is used to place a stressed emphasize on text.

  • <i>: is used to convey text in an alternative voice or tone.

Building Structure

Overview

HTML5:

  • HTML5 introduced some new structurally based elements, which can give meaning to the organization of our pages and improve its structural semantics.

Header

  • <header>: is used to identify the top of a page, article, section, or other segment of a page.
  • This part can contain introductory text, even navigation menu.

Confusion Tips:

  • <header>: a structural element which outlines the heading of a segment of a page.
  • <head>: this element not display on a page and is used to outline metadata.
  • <h1>: is used to designate multiple levels of text headings throughout a page.

Navigation

  • <nav>: is used to identifies a section of major navigational links on a page.
  • These links include links to other pages within this website or to parts of the same web page.

Article

  • <article>: is used to identify a section of independent, self-contained element.

Section

  • <section>: is used to identify a thematic grouping of content.
  • This element is often used to break up the whole page and provide hierarchy. In addition, this element always contains a heading

Aside

  • <aside>: is used to hold contents, such as sidebars or brief explanations.
  • This element is often used to identify content related to the article, such as author etc.

Footer

  • <footer>: is used to identify the closing or end of a page, article or section.

Encoding Special Characters:

&ndash, &mdash, &copy: this are special characters in HTML, and those are how they are encoded.

Hyperlinks

Overview

  • The hyperlinks facilitate the website to provide link which can link to many other websites or other pages or parts within this website.

  • <a href="...">: is used to create hyperlink which is a inline-level element.

Relative & Absolute Paths:

  • Relative Paths: If the link links to other pages of the same website, we can use relative paths in href.
  • Absolute Paths: If the link links to other website, we should use absolute paths in href.

Linking to an Email Address:

  • Email Link: can open the email client and write to the given email address.
  • The format looks like this:
  • mailto: we add the target address here.
  • subject: we add our title of the email here.
  • body: we add our content of the email here.
  • Remark: %20: means space, %0A: means line break

Open Links in a New Window

  • Just add target="_blank" in the <a>.

Linking to Parts of the Same Page

  • Back to Top Button: Just add id="top" to the target part and set href="#top".

#Breadth-first Search: 广度优先搜索

图（Graph）

  • 定义：图由节点和边组成，相邻的节点称为邻居。
  • 有向图：边带有箭头，表示单向关系（e.g. A——>B, A是B的邻居，而B不是A的邻居）

  • 无向图：边不带箭头，表示双向邻居关系（e.g. A——B）

  • 代码实现：将节点映射到其的邻居——散列表

广度优先搜索（Breadth-first Search）

  • 问题背景：通常用于解决最短路径问题，可以用于回答两类问题：

    • 1.从节点A出发，是否存在前往节点B的路径
    • 2.从节点A出发，前往节点B的哪条路径最短

  • 特点：广度优先搜索的搜索范围从起点开始逐步向外延伸，先检查一度关系，再检查二度关系，依此类推。因此，找到的路径一定是最优的

  • 算法思路：

    待搜索集S，初始节点T0，已探索集K

    1. T0的邻居节点加入S
    2. 当S不为空， 取出一个节点，判读是否为终节点
      • 是：搜索结束（找到目标）
      • 否：将该节点放入K，此后不再搜索该节点，并将该节点的邻居节点放入S
    3. 当S为空，搜索结束（未找到目标）

  • *注：加入已探索集的目的是防止邻居节点构成闭环，导致无限循环。

队列

  • 特点：一种先进先出(FIFO)的数据结构，能够实现按照元素的添加顺序进行检查，符合广度优先搜索（Breadth-first Search）的要求。
  • 操作：包含入队和出队两种操作

#Hash Table: 散列表

散列函数

  • 定义：散列函数将输入映射到数字

  • 特点：

    • 相同的输入返回同一个数字（一致性）
    • 不同的输入映射到不同的数字

Python中的散列表实现

散列表的应用

  • 创建映射&查找

  • 防止重复

  • 用作缓存

冲突

  • 冲突的含义：给两个键分配相同的位置称为冲突
  • 冲突的解决：若两个键映射到同一位置，则在该位置使用链表
  • 经验：
      1. 应设计好散列函数，将所有的键均匀的映射到散列表的不同位置
      2. 若散列表存储的链表很长，散列表的速度将急剧下降

性能

散列表平均情况 散列表最糟情况 数组 链表
查找 O(1) O(n) O(1) O(n)
插入 O(1) O(n) O(n) O(1)
删除 O(1) O(n) O(n) O(1)
  • 评价：平均情况下，散列表兼具数组和链表的优点，但这要求两点：
      1. 较低的填装因子
      2. 良好的散列函数

填装因子

  • 定义：散列表中包含的元素数 / 位置总数
  • 影响：当装填因子增大时，散列表中的元素可能不再均匀分布，这时需要调整散列表的长度（Resize）——创建一个长为原数组两倍的数组，使用散列函数将所有元素插入到新的散列表中。
  • 经验：当装填因子>0.7时，便需要跳针散列表的长度

良好的散列函数

  • 影响：良好的散列函数时数组中的值均匀分布而非大量冲突
  • 例子：SHA函数
